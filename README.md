# 📊 Vendor Performance Analysis

## Project Overview
This project focuses on analyzing vendor-level sales and purchase data to uncover key business insights related to revenue contribution, profitability, and vendor performance. The goal is to identify high-performing vendors, detect inefficiencies, and support data-driven decision-making.

## Tools & Technologies
- Python (Pandas, NumPy, Matplotlib, Seaborn)
- SQL (Data extraction & transformation)
- Power BI (Dashboard & visualization)

## Objectives
- Identify top vendors contributing to total purchases
- Analyze vendor-wise profitability and margins
- Detect high-revenue but low-profit vendors
- Evaluate pricing trends across order sizes
- Assess vendor dependency using Pareto analysis (80/20 rule)

## Dataset Description
The dataset consists of multiple tables:
- Sales data
- Purchase data
- Vendor invoices
- Product pricing
- Begin Inventory
- End Inventory
  
These datasets were merged and processed to create a unified analytical dataset.

## Key Insights
- ~65% of total sales come from top vendors → High dependency risk
- Significant capital is tied up in unsold inventory → Cash flow issue
- High-margin products have low sales → Opportunity for promotion
- Bulk purchasing reduces unit cost by ~74% → Economies of scale
- Inventory turnover varies widely → Inefficient demand planning

## Business Recommendations
- Diversify vendor base to reduce dependency
- Optimize inventory using demand forecasting
- Promote high-margin, low-sales products
- Improve pricing strategies
- Control logistics and freight costs

## Conclusion
This project demonstrates how data analysis can be used to:
- Improve vendor decision-making
- Optimize inventory management
- Increase profitability
- Reduce operational risks
